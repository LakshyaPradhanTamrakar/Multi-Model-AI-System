
Deepening LLM Theory: Looked into embeddings (how text turns into math vectors so words with similar meanings sit closer together) and context windows (the strict limit on how much text a model can read at once).

Prompting vs Fine-Tuning: Checked out how phrasing changes results (zero-shot vs few-shot prompting), the difference between actually updating model weights (fine-tuning) versus just giving instructions, and major limits like hallucinations.

The Text Pipeline: Tracked the exact step-by-step workflow of how text travels through tokenization, gets turned into embeddings, runs through the model, and comes back out as words.

Coding from Scratch: Built a basic n-gram language model to see how next-token prediction works on a small scale, and wrote a toy tokenizer to see exactly how raw text gets chopped up before a model processes it.
